Scanner Engine — núcleo de escaneo
Core
Async multi-hiloProgress en vivoProfundidad configurableArchivos ocultosSymlinks
  • Traversal recursivo desde el directorio elegido por el usuario hasta cualquier profundidad, incluidos archivos y carpetas ocultos (prefijo punto, atributos NTFS hidden/system)
  • Barra de progreso Rich en CLI con leyenda activa del archivo siendo escaneado, ETA dinámica y contador de hallazgos en vivo
  • En GUI: barra QProgressBar actualizada por QThread worker sin freeze de UI, con el path activo en un QLabel debajo
  • Modo turbo (solo extensiones de riesgo) y modo profundo (todos los archivos, análisis de entropía byte a byte)
  • Exclusión inteligente de directorios de sistema de Windows (WinSxS, $Recycle.Bin) salvo override explícito del usuario
  • Soporte de unidades de red mapeadas (\\UNC paths) y detección de permisos insuficientes con log de errores de acceso
  • Pausa / Resume del escaneo con persistencia del estado en SQLite para continuar tras reboot
  • Checkpoint automático cada N archivos para recuperación ante crash

Pattern Engine — detección multicapa
Avanzado
YARA rulesRegex libraryEntropy analysisSQLite parserJSON/YAML deep scan
  • YARA rules actualizables: detecta patrones binarios de claves privadas, certificados, blobs cifrados y formatos propietarios
  • Biblioteca de ~200 expresiones regulares para: API keys (AWS, GCP, Azure, OpenAI, Stripe, Twilio…), JWT (Header.Payload.Sig), OAuth tokens, GitHub/GitLab PAT, npm tokens, Docker auth, SSH private keys (RSA/Ed25519/ECDSA), PEM blocks, connection strings de DB, .env vars
  • Análisis de entropía Shannon: blobs con entropía > 7.2 bits/byte se marcan como potencialmente cifrados o codificados en base64 con secretos embebidos
  • Parser nativo de SQLite para leer cookies browsers (Chrome/Edge/Firefox/Brave/Opera) directamente desde los .sqlite sin copiar el archivo
  • Deep scan de JSON/YAML/TOML: traversal de árbol buscando keys sospechosas (password, secret, token, api_key, private_key, credential, auth…)
  • Detección de archivos de configuración de IDEs: VS Code (keytar, globalStorage, token), JetBrains (idea.key, passwords.xml)
  • Parser de formatos de credenciales Git: .git-credentials, .netrc, git config con insteadOf, GitHub CLI hosts.yml
  • Nivel de confianza por hallazgo: HIGH (match exacto de patrón conocido), MEDIUM (entropía + contexto), LOW (nombre de variable sospechoso)

App Fingerprinting — identificación de software
Inteligente
Path signaturesPID → process mapRegistry lookup+40 apps reconocidas
  • Base de datos de firmas de path para +40 apps de desarrollo y productividad: VS Code, Git, GitHub CLI, Antigravity, Node/NPM, Docker, AWS CLI, GCloud, Azure CLI, Terraform, Ansible, kubectl, JetBrains suite, Postman, Insomnia, DBeaver, TablePlus, Sourcetree, Fork, Figma, Slack, Zoom, Chrome, Firefox, Edge, Brave…
  • Para cada archivo sensible encontrado: mapeo inverso path → nombre de app responsable → versión detectada (desde registry o executables)

  • Detección de apps que usan Windows Credential Manager para sus credenciales
  • Iconos de app en el dashboard GUI para identificación visual instantánea
  • Flag de app activa: si la app está corriendo en el momento del escaneo, advierte antes de cualquier operación sobre el archivo

Vault Module — cifrado y gestión segura
Seguridad
AES-256PBKDF2 + saltOne-click encryptBatch opsMaster key
  • Cifrado AES-256-GCM con clave derivada de master password via PBKDF2-HMAC-SHA256 (600k iteraciones), salt único por sesión
  • One-click encrypt desde cualquier hallazgo: el archivo original se reemplaza con su versión cifrada (.ds-vault) manteniendo el path intacto
  • One-click decrypt: restaura el archivo en su ubicación original con permisos preservados
  • Batch encrypt/decrypt: opera sobre toda la lista de resultados o una selección filtrada con una sola acción
  • Planificador integrado: cifrar automáticamente al apagar Windows, descifrar al iniciar sesión (integración con Task Scheduler)
  • Registro de Vault: qué archivos están cifrados, cuándo, hash SHA-256 del original para verificación de integridad al descifrar
  • Protección del master key con Windows DPAPI como capa adicional (opcional)
  • Modo "Panic": cifra todo el vault instantáneamente con hotkey configurable

Risk Scoring Engine — puntuación de riesgo
Scoring
Score 0-100Weighted factorsRisk matrixPrioritización
  • Cada hallazgo recibe un score de 0-100 basado en: tipo de secreto (SSH key = 95, cookie de sesión = 75, API key = 85…), contexto (plaintext = más alto, dentro de .git = más alto), antigüedad del archivo, si hay proceso activo usándolo
  • Matriz de riesgo: CRITICAL (score 85-100) → HIGH (65-84) → MEDIUM (40-64) → LOW (0-39)
  • Risk summary en dashboard: total de hallazgos, breakdown por nivel, archivos únicos afectados, apps involucradas
  • Sugerencias de acción automáticas según el tipo y score: "Revoca este token en GitHub", "Este archivo debería estar en .gitignore", "Añade al Vault para cifrar al finalizar el día"

Admin Elevation & Privilegios
Windows UAC
UAC auto-elevatectypes ShellExecuteManifest runasToken check
  • Arranque con verificación automática de permisos de administrador via ctypes (IsUserAnAdmin)
  • Si no es admin: auto-relanzamiento con ShellExecute runas mostrando el diálogo UAC de Windows antes de cualquier escaneo
  • Manifest embebido en el ejecutable con requestedExecutionLevel=requireAdministrator para builds de distribución
  • Sin permisos admin: modo limitado que escanea solo el perfil del usuario actual, con advertencia clara en UI
  • Acceso a Windows Credential Manager (WCM) con pywin32 para listar credenciales almacenadas del sistema (también fuente de información sensible)
  • Acceso al registro de Windows para detectar credenciales y tokens almacenados por apps en HKCU/HKLM
